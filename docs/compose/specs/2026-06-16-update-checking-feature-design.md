# [S1] Problem

用户需要一种手动检查应用程序更新的方式。当前应用程序有自动更新基础设施，但缺少用户界面来触发检查更新和显示更新详情。

# [S2] Solution overview

在标题栏中添加自定义菜单面板，集成检查更新功能。菜单栏图标显示更新状态（小圆点或徽章）。点击图标展开菜单，包含检查更新、更新详情等选项。同时在状态栏中添加更新状态图标。

# [S3] Architecture

1. **主进程**：扩展现有的`updater.ts`，添加更多IPC处理器
2. **预加载脚本**：扩展现有的update API，添加更多方法
3. **渲染进程**：创建`UpdateMenu.vue`组件，修改`TitleBar.vue`和`StatusBar.vue`

# [S4] Components

1. **UpdateMenu.vue**：自定义菜单面板组件
   - 检查更新按钮
   - 更新状态显示
   - 更新详情（版本号、大小、更新日志）
   - 下载/安装按钮

2. **TitleBar.vue修改**：添加更新图标和菜单触发器
3. **StatusBar.vue修改**：添加更新状态图标

# [S5] Data flow

1. 应用启动时自动检查更新
2. 用户点击检查更新按钮触发手动检查
3. 检查到更新时菜单栏图标显示更新标记
4. 用户点击图标查看更新详情
5. 用户选择下载/安装

# [S6] Error handling

1. 网络错误时显示友好提示
2. 下载失败时提供重试选项
3. 安装失败时提供手动下载链接
4. 后台静默检查更新，避免影响用户体验

# [S7] Testing

1. 单元测试：更新检查逻辑、状态管理
2. 集成测试：IPC通信、组件交互
3. 手动测试：完整更新流程、网络异常场景
